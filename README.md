# parsing-astroexpert

Скрипт для парсинга страницы Astro.Expert:

- данных из таблицы планет (`table.planets`);
- координат планет из SVG-карт (`svg.render`);
- дома планеты относительно `Asc`;
- позиции планеты в D1-сетке (ячейка `1..16`, `row/col`, `x/y`).

## Файл

- `parcing_code` — JS-скрипт для запуска в консоли браузера на странице карты.

## Что возвращает скрипт

Скрипт печатает в консоль два объекта:

1. `D1 + таблица` — массив планет с полями:
   - `planet`, `karaka`;
   - `sign`, `degree`;
   - `nakshatra`, `pada`;
   - `extra`;
   - `house` (дом от Asc, 1..12);

   - `retrograde` (`true/false`, если в SVG подпись планеты в скобках, например `(Sa)`);

   - `d1Cell`, `d1Row`, `d1Col`, `d1X`, `d1Y`.

2. `Все найденные карты (D1/Dx 500x500)` — массив карт на странице:
   - `chartIndex`;

   - `chartName` (например: `тест (D1)`, `Навамша (D9)`, `Дашамша (D10)`);
   - `viewBox`;
   - `ascCell` (ячейка Asc, взятая из стрелки на карте);
   - `planets` с координатами по каждой карте + `sign` (фиксированный знак ячейки)
     и `house` (номер дома от `ascCell`).


## Как использовать

1. Откройте нужную карту в Astro.Expert.
2. Откройте DevTools (`F12`) → `Console`.
3. Вставьте содержимое файла `parcing_code`.
4. Нажмите `Enter`.
5. В консоли появятся оба результата.

## Предположения и ограничения

- D1 распознается по `viewBox="0 0 600 600"`.
- Дробные карты распознаются по `viewBox="0 0 500 500"`.
- Используется южный стиль с сеткой `4x4`.
- Берутся только темные (`text-gray-900`) подписи планет в SVG.

- Маркеры рядом с планетой (например, `↑` / `↓`) очищаются при нормализации имени:
  `Me↑` и `Me↓` парсятся как `Me`.
- Для вычисления `house` используются русские названия знаков:
  `Овен ... Рыбы`.
